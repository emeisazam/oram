oram

a speech-operated terminal looper for synthetic sound studies.

record sound into four layers. shape it with dsp transforms. summon textures from prompts. let the agent listen and react.

workflow

  1. pick a layer (1–4) and press ⏺ to record from the mic.
  2. type into the prompt below the layers to describe a texture; press ↵ to summon it into the selected layer.
  3. drag across a waveform to set a loop region. click it to clear.
  4. open fx ▾ to transform the selected layer.
  5. switch input mode in the header: prompt · audio · listen.

per-layer corners

  • top-left · select layer · click again = mute · right-click = solo
  • top-right · export / bounce to WAV (green)
  • bottom-left · auto-generate from what's sounding (purple)
  • bottom-right · clear / delete layer audio (red)
  • drag waveform · set a loop region · click waveform · clear loop
  • vol strip · drag, scroll, dbl-click = unity

keyboard

  • 1–4 select layer
  • m mute selected · u unmute all
  • g generate · l listen
  • r record · o overdub · k kill all
  • ⌘K or ? command palette · / focus prompt · esc close

bring your own keys

your keys stay on your machine. oram reads them locally and never uploads them.

  1. copy .env.example to .env in the oram folder.
  2. paste any provider keys you have — keep only the lines you use: 
    ELEVENLABS_API_KEY=sk_...
FAL_KEY=...
STABILITY_API_KEY=sk-...
HF_TOKEN=hf_...
REPLICATE_API_TOKEN=r8_...
  3. restart oram. open ⚙ in the header to pick a specific engine, or leave on auto.

without keys, oram runs in mock mode — fine for learning the interface, but it won't generate real audio. add at least one provider key to summon textures from prompts.

local providers (TangoFlux, Kokoro, Whisper.cpp, Essentia) run on your machine and need no key. custom HuggingFace endpoints are reached via HF_TOKEN — paste your token and use a hosted endpoint URL in your generator config.
